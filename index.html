批判的実在論の特徴

技術システムの表面的問題から深層の社会構造的問題への遡及的分析を可能にする。
技術的「問題解決」を社会的「構造変革」へと転換しながらも、認識プロセスの基本構造は保存される。

埋め込みの特性

  • 象限：Its (集合外面) - 客観的で観察可能な技術システム領域
  • 証拠の性質：定量的・測定可能・検証可能な技術データ
  • 主体の性質：役割として定義された利害関係者(職務上の立場)

ステップ2: 象限変換 (Transformation)

変換: M⁶⁵|Its → M⁶⁵|(We∪Its)
変換される要素と保存される構造
要素 Its象限での表現 (We∪Its)象限への変換 変換の性質
Observer システム利害関係者
(役割)		 社会変革行為主体
(Agency)		 技術的役割→社会的行為主体へ
Focus 技術的制約・要求 社会機構・権力構造 技術的問題→社会構造的問題へ
Method システム設計視点 変革条件・介入戦略 技術的方法→社会変革方法へ
Construction 設計成果物(ビュー) 実際的事象・社会変化 技術的成果物→社会的変革実践へ
Target 目標システム 実在構造・深層変革 技術的システム→社会的実在へ
Data 技術仕様書・評価結果 経験データ・観察現象 技術的データ→社会的経験データへ

✓ 保存される構造

  • 6要素間の射の合成関係: attend ∘ select ∘ apply ∘ aim ∘ evidence ∘ learn
  • 循環的フィードバック構造: 社会変革の循環的プロセス
  • 要素間の依存関係: Mechanismの理解なくしてCondition(変革戦略)は選べない

✗ 意図的に変化する性質

  • 視点: 純技術的(Its) → 社会文化的+技術的(We∪Its)
  • 存在論: 技術的実在 → 批判的実在論的三層構造
  • 目的: システム最適化 → 社会変革と解放
  • 証拠の性質: 技術的測定データ → 社会現象の経験的データ

ステップ3: 射影 (Projection)

π_CR: M⁶⁵|(We∪Its) → 批判的実在論
M⁶⁵座標から批判的実在論の用語体系への再解釈
M⁶⁵|(We∪Its)の座標 批判的実在論の概念 具体例
Observer@(We∪Its) Agency
(個人・集合行為主体)		 社会的抑圧構造を認識し変革しようとする研究者・活動家・コミュニティ
Focus@(We∪Its) Mechanism
(社会機構・権力構造)		 技術システムに埋め込まれた権力関係・格差構造・排除のメカニズム
Method@(We∪Its) Condition
(変革条件・介入戦略)		 社会構造を明らかにし変革するための批判的方法論
Construction@(We∪Its) Actual
(実際的事象・社会変化)		 実際に起きている社会的出来事・技術システムの社会的帰結
Target@(We∪Its) Real
(実在構造・深層変革)		 目に見えない深層の社会構造・メカニズム・因果的力の変革
Data@(We∪Its) Empirical
(経験データ・観察現象)		 人々が経験する社会現象・技術システムの影響の観察データ

批判的実在論の三層ドメイン構造

Real Domain (実在的ドメイン)
目に見えない深層構造・メカニズム・因果的力 → Targetに対応
Actual Domain (実際的ドメイン)
実際に起きる出来事・社会変化 → Constructionに対応
Empirical Domain (経験的ドメイン)
観察可能・経験可能な現象 → Dataに対応

この層構造により、技術システムの表面的問題から深層の社会構造的問題への遡及的分析が可能になる

具体的翻訳例

IEEE 42010での記述:

"Stakeholder 'システム運用者' has Concern 'システムの信頼性向上' and designs System 'クリティカルインフラシステム' through Viewpoint 'セキュリティ設計視点'"

↓
埋め込み後 M⁶⁵|Its座標:

Observer=システム運用者(役割), Focus=技術的信頼性要求, Method=標準的セキュリティ設計手法, Target=信頼性の高いシステム, @Its象限

↓
象限変換後 M⁶⁵|(We∪Its)座標:

Observer=社会的行為主体, Focus=技術システムに埋め込まれた権力構造, Method=批判的分析と変革戦略, Target=より公正な社会構造, @(We∪Its)象限

↓
射影後 批判的実在論での記述:

"Agency 'システム設計に関わる研究者・技術者' は Mechanism 'クリティカルインフラに埋め込まれたアクセス格差・技術的排除の構造' を認識し、Condition '批判的システム設計方法論' を用いて、Actual '現実の技術システムにおける不平等' を分析し、Real '深層にある社会構造的不正義の根本原因' を明らかにして変革する。Empirical '人々が経験する技術的排除やアクセス困難' というデータに基づく"

保存された構造

  • 射の合成則: 順序関係の保存
  • 循環構造: 社会変革の循環的プロセス
  • 要素間依存関係: 因果的理解の構造

意図的に変化した要素

  • 存在論: 技術的実在 → 三層構造
  • 目的: 最適化 → 変革と解放
  • 主体: 技術的役割 → 社会的エージェンシー
5/5
